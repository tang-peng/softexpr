实验3-1d. 定时器
  • MFC定义了定时器类CTimer，可定时触发事件响应函数，常用于周期性地状态更新。

需求说明：

  1. 在基于对话框的MFC应用中，添加定时器，使程序在执行时每1秒钟能够用TRACE输出一条消息。

实验结果:

  1. 给出程序执行时TRACE动态输出消息的截图。
  2. 总结SetTimer、OnTimer和KillTimer函数的用法。

提示：

  • 首先，在向导自动生成的代码中找到CDialog的初始化函数OnInitDialog，在其中使用SetTimer()函数，以创建定时器。

    xxxxxxxxxx
    6
    1
    BOOL CMFCApplicationDlg::OnInitDialog() {
    2
        ...
    3
        // 在此添加额外的初始化代码
    4
        SetTimer(1, 100, NULL); // 参数1：定时器标记，参数2：间隔时间
    5
        ...
    6
    }

    注意：一般将代码添加到函数末尾处，此处通常有// 在此添加额外的初始化代码的提示。

  • 其次，通过类向导(Class Wizard)添加OnTimer(nIDEvent)消息处理函数。

       

    该函数中的代码将在定时器被触发时自动运行。

    xxxxxxxxxx
    5
    1
      void CMFCApplicationDlg::OnTimer(UINT_PTR nIDEvent) {
    2
          // TODO: Add your message handler code here and/or call default
    3
          CDialogEx::OnTimer(nIDEvent);
    4
          TRACE(_T("Timer triggered\n"));
    5
      }

    如果程序中有多个定时器，则共用一个OnTimer消息处理程序，并通过判断定时器标志nIDEvent来区分不同的定时器。

  • 最后，当不需要计时器的时候调用KillTimer(nIDEvent)。

附件3-1 VS2013下创建MFC工程

使用MFC应用向导(MFC Application Wizard)

  • 打开VS2013(VS201X界面类似)，在菜单中选择：FileNewProject。

  • 选择MFC Application，并适当命名，然后点OK。

  • 在Application type选项卡中选择类型，建议Project style选择MFC standard，然后点击Finish以结束向导，或者点Next以进一步设置。

  • 点击Local Windows Debugger(绿色小三角图标)，以编译并运行程序。

  • 运行效果。

附录3-2 添加按钮

使用资源视图(Resource View)

  • 首先新建基于对话框的MFC应用，然后需打开Resource View(资源视图)，找到对话框以添加控件。打开资源视图的步骤是：选择菜单View$Resource View(各版本VS稍有不同)。

  • 在资源视图中，找到Dialog下的IDD_MFCAPPLICATION_DIALOG(名字可能稍有不同)，然后双击，以打开对话框界面。

  • 在界面中添加Toolbox(控件工具栏)，以便于添加控件。

添加按钮(Button)

  • 从Toolbox中拖拽Button到对话框。

  • 在按钮上点右键，选择Add Event Handler。

  • 设置响应函数的名称，并选Add and Edit。

  • 在出现的函数中，添加代码。请注意，该函数名即为前一步所设置的函数名。

  • 编译运行，可看到TRACE输出结果到output窗口中。

    注意，该函数的缺省形式是void CMyAppView::OnDraw(CDC* \*pDC*\)，即参数pDC被注释掉了。去掉注释符即可。

  • 对话框应用中，重载CDialog::OnPaint()函数实现绘图，并且需用编写代码来获取CDC指针。
